# Changelog

## 0.7.65

- 修复智谱清言（GLM）批量导出时，`chatstash_title` 合成会话 URL 被误判为当前页 `already` 的问题。
- 现在 GLM 会优先按 `cid` 或侧栏选中标题判断是否已在目标会话中，避免根本没切换会话就直接开始提取。
- 同步统一 UI 层的 GLM 会话 ID 识别，减少“仅当前页”与实际导出导航判断不一致的情况。

## 0.7.64

- 修复智谱清言（GLM）导出时将“来自：智谱清言”“言”等页头碎片误判为有效对话内容的问题。
- GLM 在提取到无效碎片时不再直接导出，而是继续走重试流程；若始终未拿到有效正文，则返回失败而不是生成错误的 Markdown/PDF。
- 移除 GLM 回退提取中将会话标题伪造为用户消息的逻辑，避免导出结果看似完整、实际内容失真。

## 0.7.63

- 修复智谱清言（GLM）批量导出时可能误抓首页空状态文案，避免 Markdown 中出现“最新旗舰模型上线”“和我聊聊天吧”等非会话内容。
- 收紧 GLM 切换会话后的内容就绪检测，只有真正会话正文出现后才开始提取，减少批量导出抢跑导致的空白或错页问题。

## 0.7.61

- 为智谱清言（GLM）已抽取的消息 turn 增加统一内容净化，确保正常抽取和回退抽取都能过滤相同噪音。
- 继续过滤页头工具条、页码 `1/1`、活动图和 AI 页脚标记，减少 Markdown 中不属于正文的残留内容。

## 0.7.60

- 继续清理智谱清言（GLM）导出中的页头短噪音，过滤 `ChatGLM...语音`、`梦幻杰` 等工具条/入口文本。
- 在回退抽取中跳过正文开始前的空白行和短噪音行，减少 Markdown 开头残留杂项。

## 0.7.59

- 过滤智谱清言（GLM）导出中来自 `sfile.chatglm.cn/activeimg/...` 的活动图片噪音。
- 过滤 `用户_xxx` 这类页头用户标识，减少 Markdown 开头的无意义杂项。

## 0.7.58

- 进一步收紧智谱清言（GLM）回退抽取逻辑，过滤智能体卡片、推广图片、页脚协议区和 `data:image` 噪音。
- 在 GLM 回退抽取中优先把首个明显的提问块识别为用户消息，减少整页内容被错误并入 Assistant 的情况。
- 修复生成式 PDF 在粗体字体回退时可能出现的中文编码报错。

## 0.7.57

- 修复智谱清言（GLM）导出 Markdown 时只抽取到零碎节点的问题，新增正文块级回退提取逻辑。
- 智谱清言（GLM）PDF 导出改为使用插件内生成式 PDF，不再依赖网页打印，避免首屏空白、输入框和底部操作区被带入。
- GLM 导出文件名优先使用侧栏会话标题，避免批量导出时反复出现通用标题 `智谱清言`。

## 0.7.56

- 优化智谱清言会话列表展示，标题优先显示，并默认隐藏 `chatstash_title` 形式的合成链接。
- 在面板标题栏显示已读取的会话数量，便于确认侧栏扫描结果。

## 0.7.55

- 将“侧栏会话列表读取”纳入调试日志，支持定位智谱清言会话列表为空或标题异常的问题。
- 让面板读取列表与导出流程共用同一份日志，避免未进入导出阶段时日志为空。

## 0.7.54

- 为智谱清言导出增加切换会话后的额外等待与重试，减少内容尚未刷新完成时的读取失败。
- 扩大 GLM 消息节点候选范围，并在提取失败时记录节点样本，便于继续定位页面结构差异。

## 0.7.53

- 改为直接读取智谱清言 `#aside-history-list .list .history-item .title`，不再依赖泛化的侧栏猜测逻辑。
- 修复 GLM 批量导出时按标题点击会话项的导航方式，与页面实际 DOM 结构对齐。
- 补充 GLM 当前会话识别逻辑，使“仅当前页”能正确选中已激活的历史项。

## 0.7.52

- 增加智谱清言（GLM）按标题识别会话的兜底逻辑，兼容无链接、无显式 `cid` 的历史列表。
- 为 GLM 批量导出补充按标题点击侧栏会话项的导航方式。
- 同步更新扩展版本号与发布说明。

## 0.7.51

- 增强智谱清言（GLM）会话列表读取，新增对非链接历史项的识别。
- 为 GLM 批量导出补充 `cid` 节点点击逻辑，兼容 `div/li/button` 形式的侧栏会话项。
- 同步更新扩展版本号与发布说明。

## 0.7.50

- 优化智谱清言（GLM）会话列表读取逻辑，支持自动滚动侧栏收集更多历史会话。
- 保留当前会话回退逻辑，避免侧栏暂未加载时完全读不到会话。
- 同步更新扩展版本号与发布说明。

## 0.7.49

- 新增智谱清言（GLM）初版网页适配，支持当前会话提取与可见侧栏会话列表加载。
- 为 `chatglm.cn/main/alltoolsdetail` 增加 `cid` 导航标识，修复批量导出时不同会话 URL 无法正确区分的问题。
- 扩展注入范围新增 `chatglm.cn`，为后续继续完善 GLM 导出稳定性打基础。

## 0.7.48

- 清理 Kimi 导出中的 OCR 提示文案，避免正文被遮挡并混入导出结果。
- 调整 Kimi 会话列表策略，优先使用当前页面可见会话，减少批量导出时因侧栏不可见导致的失败。
- 补充 Kimi 会话内容加载检测，提高切换会话后的导出稳定性。

## 0.7.47

- 修复 Kimi PDF 只导出单页的问题，改为使用 Kimi 专用打印样式展开聊天滚动区域。
- 同步规范版本号管理，确保每次修改都更新版本号与更新说明。

## 0.7.46

- 新增 Kimi 网页适配。
- 优化 Kimi 会话列表加载与消息提取稳定性。
- 修复扩展在部分页面不显示入口的问题。
